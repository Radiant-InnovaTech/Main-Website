Architecture

Telemetry in. Reasoned response out.

STREAMING

Stage by stage.

What happens between signal and resolution.

1
Telemetry Ingest Ingest

Raw event streams from EDR, cloud audit logs, identity providers, and network sensors flow into the ingest layer over encrypted transport.

Input: raw events from across your estate
2
Normalisation Parse

Every event is parsed and typed against the Open Cybersecurity Schema Framework (OCSF), producing a consistent record structure regardless of source vendor.

Schema: typed OCSF records
3
Correlation Synthesise

Graph-based clustering groups semantically related events across entities and time windows, collapsing alert storms into structured investigation threads.

Compression: 150:1 alert-to-incident
4
Reasoning Core Reason

The orchestrating intelligence layer. Decomposes the incident context, delegates sub-tasks to specialised agents, aggregates confidence scores, and determines whether autonomous action or human escalation is appropriate.

5
Investigation Graph Map

A full attack timeline is assembled with lateral movement paths, entity relationships, and MITRE ATT&CK mappings — built automatically from correlated evidence.

Assembly: automatic, in < 60 seconds
6
Response Orchestration Respond

Containment playbooks are compiled and staged with rollback verification. Execution is gated behind analyst approval — no action runs unless a human approves it.

Safety: rollback-verified, state-preserving
7
Persistent Memory Remember

Every observation, pattern, and outcome is written to an encrypted embedded vector store — retained indefinitely and queryable by all agents across future investigations.

Store: encrypted embedded vector memory
8
Analyst Oversight Supervise

Analysts retain complete visibility and control. Every agent decision is traceable, every action approved, every deviation from expected behaviour is flagged with full evidence context.

Trace: complete auditability and provenance

The same boundary guarantees.

All four deployment models share identical security, isolation, and auditability properties.

Tenant Isolation

Every customer's data, memory, and agent state is cryptographically isolated. No cross-tenant data access is architecturally possible.

Supervised Action

All containment and remediation actions are staged behind human approval gates. The platform can never take autonomous destructive action without explicit analyst sign-off.
